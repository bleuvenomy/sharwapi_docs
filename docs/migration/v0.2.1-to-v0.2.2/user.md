# 从 v0.2.1 升级到 v0.2.2 — 用户侧

本页面面向**部署和运维 SharwAPI 主程序**的用户，介绍从 v0.2.1 升级到 v0.2.2 时需要关注的破坏性变更。

如果你也是插件开发者，请另行阅读 [开发者侧迁移指南](./developer)。

## 破坏性变更

### 插件目录重命名：`Plugins/` → `plugins/`

主程序扫描的插件目录已由 `Plugins` 重命名为 `plugins`（首字母改为小写）。

**影响范围**

| 环境 | 影响 |
|---|---|
| **Linux / macOS** | ⚠️ **受影响**：文件系统大小写敏感，需手动重命名目录，否则升级后所有插件无法加载 |
| **Windows** | ✅ 不受影响：文件系统大小写不敏感，无需任何操作 |

**Linux / macOS 用户的必要操作**

在替换主程序二进制文件之前，先将现有的插件目录重命名：

```bash
mv Plugins plugins
```

完成后，重新启动新版主程序即可。

## 升级步骤总览

1. （**Linux / macOS**）将插件目录从 `Plugins/` 重命名为 `plugins/`
2. 用新版主程序替换旧版可执行文件
3. 启动新版主程序，观察日志确认插件正常加载
